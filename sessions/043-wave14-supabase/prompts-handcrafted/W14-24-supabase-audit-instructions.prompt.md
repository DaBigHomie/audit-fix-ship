# Supabase Audit Invariants (post-Wave-14)

## 1. Single Canonical Types File
- Canonical: `src/lib/supabase/types.ts` (CI-regenerated by `supabase-migration-automation.yml`)
- ❌ NEVER recreate `src/shared/types/database.ts`
- ❌ NEVER hand-edit `src/lib/supabase/types.ts` (regen only)

## 2. Single Canonical Client Wrappers
- Browser: `import { createBrowserClient } from '@/lib/supabase/client'`
- Server: `import { createServerClient } from '@/lib/supabase/server'`
- ❌ NEVER `import { createClient } from '@supabase/supabase-js'` outside `lib/supabase/`
- `import type` from `@supabase/supabase-js` is allowed (type-only)
- See `.github/instructions/supabase-clients.instructions.md` for the ESLint guard

## 3. Image Origins (post-Wave-14)
- Canonical origin: **Supabase Storage CDN** (decision Q2)
- Brand-chrome exception list (Q5): logo*, favicon*, og-default*, twitter-card*, manifest-icon-* — these stay in `public/`
- Every other image MUST resolve through `media_library` table (Q7)

## 4. Audit Script Behavior
- `scripts/audit-product-images.mts` is CI-blocking (`CI=true` → `exit 1` on violations)
- Locally, same script warns (does not exit non-zero)
- See `scripts/audit-product-images.mts` and W14-09

## 5. Migration Conventions
- Apply via `node scripts/apply-migration-via-rest.cjs <file>` (NOT `supabase db push`)
- Backup first via `npx tsx scripts/backup-supabase-state.mts <scope>` for any DROP/TRUNCATE/DELETE
- Workflow `supabase-migration-automation.yml` hard-blocks PRs with destructive ops without explicit override label

## 6. RLS Coverage
- Every public table MUST have RLS enabled
- Every RLS policy MUST be reviewed in PR
- Service-role keys NEVER in client code (only `lib/supabase/server.ts`)

## 7. Secret Naming (UGWTF convention)
- `SUPABASE_URL_043`
- `SUPABASE_SERVICE_ROLE_KEY_043`
- Client secrets `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` are separate (browser-safe)

## 8. Pre-PR Checklist
- [ ] `npx tsc --noEmit` clean
- [ ] `npx tsx scripts/audit-product-images.mts` clean (locally — no warnings)
- [ ] `npx tsx scripts/supabase-type-sync.mts` clean (no path errors)
- [ ] No new `import { createClient } from '@supabase/supabase-js'` outside `lib/supabase/`
- [ ] No new file under `src/shared/types/database.ts`
- [ ] No new local `/images/...` reference in source (use Supabase Storage URL)

## 9. Drift Detection Triggers
If you see ANY of these in a diff, treat as P0:
- Re-creation of `src/shared/types/database.ts`
- Re-creation of `src/shared/lib/supabase-{browser,server}.ts`
- New `import { createClient } from '@supabase/supabase-js'` outside `lib/supabase/`
- Hardcoded `/images/...` in `src/`, `lib/`, `scripts/` (outside brand-chrome list)
